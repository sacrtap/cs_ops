**Purpose:** 架构关注点、可测试性差距和 NFR 要求，供架构师/开发团队评审。作为 QA 与工程团队之间的契约，明确测试开发前必须解决的问题。

## Executive Summary

**Scope:** 系统级测试架构设计，覆盖 8 个 Epic、58 个用户故事的完整测试策略

**Business Context** (from PRD):

- **Revenue/Impact:** 管理 1320 个客户数据，年消费规模分级（S 级≥100 万），目标降低客户流失率≥15%，提升运营效率≥70%
- **Problem:** 客户数据分散在 Excel 中、每月 3 天手工生成结算单、193 个僵尸账号未被识别、93.6% 客户未进行价值评级
- **GA Launch:** MVP 预计 3.5 个月交付（2026 年 6 月中旬）

**Architecture** (from ADR):

- **技术栈决策:** Python 3.11 + Sanic 23+ (异步 Web) + SQLAlchemy 2.0 (ORM) + PostgreSQL 18
- **前端决策:** Vue 3.4 + Arco Design 2.54 + TypeScript 5.3
- **部署决策:** Docker + Docker Compose 容器化
- **任务队列:** Celery + Redis (异步任务处理)
- **认证:** JWT Token + 4 级 RBAC (Admin/经理/专员/销售)

**Expected Scale** (from ADR):

- 客户数据：1320 个（初期），支持 10 万 + 扩展
- 并发用户：≥50 用户同时在线
- 数据导入：≥1000 条/分钟
- 查询响应：关键查询<1 秒，复杂查询<5 秒

**Risk Summary:**

- **Total risks**: 12 个
- **High-priority (≥6)**: 8 个风险需要立即缓解
- **Test effort**: ~160 个测试场景 (~6-9 周，1 名 QA 全职)

## Quick Guide

### 🚨 BLOCKERS - 团队必须决定（没有这些无法推进）

**预实现关键路径** - 这些必须在 QA 开始集成测试前完成：

1. **BLK-01: Celery 任务测试隔离机制** - 提供测试专用 Celery eager 模式配置或同步任务执行器，支持异步任务同步测试 (推荐负责人：后端团队)
2. **BLK-02: 外部 API Mock 基础设施** - 完整 mock 邮件服务、用量采集接口，禁止测试调用真实外部服务 (推荐负责人：后端团队)
3. **BLK-03: 数据权限测试支持** - 提供测试专用数据库视图或查询接口，支持断言不同角色的数据权限过滤 (推荐负责人：后端团队)

**团队需要做什么:** 在实现前完成这 3 项，否则测试开发被阻塞

### ⚠️ HIGH PRIORITY - 团队应该验证（我们提供建议，您审批）

1. **RISK-01: 权限系统缺陷导致数据泄露** - 实现 4 角色×8 模块的权限测试矩阵，每个 API 必须验证数据权限过滤 (实施阶段，Dev + QA)
2. **RISK-03: 大数据量下性能不达标** - 建立性能基准测试 (pytest-benchmark)，关键查询 EXPLAIN 分析，预置 10 万 + 测试数据集 (Sprint 1，架构师)
3. **RISK-04: JWT 令牌泄露导致未授权访问** - 强制 HTTPS，令牌短有效期（2 小时），刷新令牌机制，敏感操作二次认证 (Sprint 1，Dev)
4. **RISK-06: Excel 导入验证不足导致脏数据** - 完整验证层（必填/格式/业务规则），导入前预览确认，失败回滚机制 (Sprint 2，Dev)
5. **RISK-09: 外部 API 不稳定导致集成测试失败** - pytest-httpx 全面 mock，契约测试确保 API 兼容，失败降级策略 (Sprint 1，Dev + QA)
6. **RISK-11: 定级规则配置错误** - 规则配置验证测试，变更影响范围测试，历史数据重算测试 (Sprint 2，Dev + QA)
7. **RISK-12: 结算金额计算错误** - 双人复核机制，异常数据检测（阈值告警），结算历史不可篡改，财务对账测试 (Sprint 3，Dev + 产品)

#### 2. 需要的架构改进 (应该改变什么)

1. **JWT 令牌过期测试支持**
   - **当前问题**: 令牌刷新逻辑、过期场景难以模拟
   - **需要的变更**: 在测试配置中缩短 JWT 过期时间（如 5 分钟），提供时间戳模拟 fixture
   - **不修复的影响**: 无法测试令牌过期/刷新逻辑
   - **负责人**: 后端
   - **里程碑**: Sprint 1

2. **Redis 缓存测试隔离**
   - **当前问题**: 缓存失效、缓存穿透场景测试困难
   - **需要的变更**: 提供测试专用 Redis 实例，每个测试前清空缓存
   - **不修复的影响**: 缓存相关测试不稳定
   - **负责人**: DevOps
   - **里程碑**: Sprint 1

3. **定时任务手动触发**
   - **当前问题**: 健康度自动更新、预警推送等定时任务难以触发
   - **需要的变更**: 提供手动触发接口用于测试，或使用 freezegun 库模拟时间流逝
   - **不修复的影响**: 定时任务测试覆盖率低
   - **负责人**: 后端
   - **里程碑**: Sprint 2

4. **并发测试安全**
   - **当前问题**: 并行测试可能导致数据竞争
   - **需要的变更**: 使用 pytest-xdist 配置独立数据库 per 测试进程，或串行执行集成测试
   - **不修复的影响**: 间歇性测试失败
   - **负责人**: QA
   - **里程碑**: Sprint 1

5. **审计日志验证支持**
   - **当前问题**: 操作日志记录难以断言
   - **需要的变更**: 提供审计日志查询 API，测试中可查询并断言日志内容
   - **不修复的影响**: 无法验证审计日志完整性
   - **负责人**: 后端
   - **里程碑**: Sprint 2

### 风险缓解计划（高风险≥6）

**目的**: 所有 8 个高风险项（分数≥6）的详细缓解策略。这些风险必须在 GA 发布前解决。

#### RISK-01: 权限系统缺陷导致数据泄露 (Score: 6) - 关键

**缓解策略:**

1. 实现 4 角色×8 模块的完整权限测试矩阵（Admin/经理/专员/销售 × 客户列表/详情/新增/编辑/转移/定级/导出/系统设置）
2. 每个 API 测试必须验证数据权限过滤（销售只能看自己客户，运营可看全部）
3. 渗透测试（使用 OWASP ZAP 工具）验证越权访问防护
4. 实现数据权限测试专用 fixture，支持快速切换角色验证

**负责人:** 开发团队 + QA  
**里程碑:** Sprint 1 (权限与认证 Epic)  
**状态:** 计划中  
**验证方式:** 测试覆盖率报告显示 100% 权限场景覆盖

#### RISK-02: Celery 任务失败导致结算数据丢失 (Score: 6) - 关键

**缓解策略:**

1. Celery 任务失败自动重试机制（最多 3 次，指数退避）
2. 任务执行完整日志记录（输入、输出、异常、重试次数）
3. 失败任务告警推送（系统内通知 + 邮件推送给运维）
4. 任务幂等性测试（重复执行不产生重复数据）
5. 故障注入测试（模拟 Redis 故障、数据库故障、网络超时）

**负责人:** 后端团队  
**里程碑:** Sprint 2 (结算管理 Epic)  
**状态:** 计划中  
**验证方式:** 故障注入测试通过，失败告警 100% 触发

#### RISK-03: 大数据量下性能不达标（查询>5 秒） (Score: 6) - 关键

**缓解策略:**

1. 建立性能基准测试（pytest-benchmark），监控关键查询耗时
2. 关键查询 EXPLAIN 分析，确保使用索引
3. 索引覆盖检查（客户 ID、价值等级、状态、创建时间等查询字段）
4. 慢查询日志监控（>1 秒查询自动记录）
5. 预置 10 万 + 测试数据集进行压力测试
6. 查询超时保护（>10 秒自动终止）

**负责人:** 架构师 + 后端  
**里程碑:** Sprint 1 (基础设施)  
**状态:** 计划中  
**验证方式:** 性能测试报告显示关键查询<1 秒，复杂查询<5 秒

#### RISK-04: JWT 令牌泄露导致未授权访问 (Score: 6) - 关键

**缓解策略:**

1. 强制启用 HTTPS（禁止 HTTP 访问）
2. JWT 令牌短有效期（2 小时）
3. 刷新令牌机制（refresh token，7 天有效期）
4. 敏感操作二次认证（密码验证或 MFA）
5. 令牌绑定 IP/设备指纹（可选增强）
6. 异常登录检测（异地登录、频繁失败）

**负责人:** 后端团队  
**里程碑:** Sprint 1 (权限与认证 Epic)  
**状态:** 计划中  
**验证方式:** 安全审计通过，渗透测试无高危漏洞

#### RISK-06: Excel 导入验证不足导致脏数据 (Score: 6) - 关键

**缓解策略:**

1. Excel 导入完整验证层：
   - 必填字段验证（名称、联系方式）
   - 格式验证（邮箱格式、电话格式、手机号格式）
   - 业务规则验证（价值等级范围、设备类型有效性）
2. 导入前预览确认（显示前 10 条数据）
3. 导入失败回滚（事务保护，部分失败全部回滚）
4. 导入日志记录（成功条数、失败条数、失败原因）
5. 边界值测试（空文件、超大文件、特殊字符、格式错误）

**负责人:** 后端团队 + 前端  
**里程碑:** Sprint 2 (客户主数据管理 Epic)  
**状态:** 计划中  
**验证方式:** 边界值测试 100% 通过，脏数据 0 容忍

#### RISK-09: 外部 API 不稳定导致集成测试失败 (Score: 6) - 关键

**缓解策略:**

1. pytest-httpx 全面 mock 外部 HTTP 请求（邮件服务、用量采集接口）
2. 契约测试确保 API 兼容（Pact.io）
3. 失败降级策略（mock 数据降级、缓存降级）
4. 超时配置（30 秒超时，避免测试挂起）
5. 网络错误监控（记录所有外部请求失败）

**负责人:** 后端团队 + QA  
**里程碑:** Sprint 1 (基础设施)  
**状态:** 计划中  
**验证方式:** 契约测试报告，集成测试不依赖真实外部服务

#### RISK-11: 定级规则配置错误导致客户价值评估错误 (Score: 6) - 关键

**缓解策略:**

1. 定级规则配置验证测试：
   - 阈值边界验证（S 级=100 万、99.9999 万）
   - 规则变更影响范围测试
2. 历史数据重算测试（规则变更后批量重算 1320 个客户）
3. 规则变更审计日志（谁、何时、从什么改为什么）
4. 规则生效延迟保护（变更不是立即生效，而是下次计算时生效）
5. 双人复核机制（规则变更需经理审批）

**负责人:** 后端团队 + QA + 产品  
**里程碑:** Sprint 2 (客户价值评估 Epic)  
**状态:** 计划中  
**验证方式:** 规则变更测试 100% 通过，历史重算准确性验证

#### RISK-12: 结算金额计算错误导致财务损失 (Score: 6) - 关键

**缓解策略:**

1. 结算金额计算双人复核（系统计算 + 人工抽查）
2. 异常数据检测（阈值告警）：
   - 0 用量客户标记
   - 用量突增（>300%）标记
   - 结算金额异常（>平均值 3 倍）标记
3. 结算历史不可篡改（已生成结算单只读）
4. 财务对账测试（系统数据 vs 财务数据）
5. 结算单审核流程（运营生成 → 经理审核 → 发送客户）
6. 边界值测试（0 用量、超量、负值等）

**负责人:** 后端团队 + 产品 + 财务  
**里程碑:** Sprint 3 (结算管理 Epic)  
**状态:** 计划中  
**验证方式:** 结算准确性测试 100% 通过，财务对账 0 差异

### 假设与依赖

#### 假设

1. 假设 Celery 任务支持 eager 模式（同步执行）用于测试
2. 假设 Redis 可用于测试环境（缓存相关测试）
3. 假设外部 API 支持 mock（pytest-httpx）
4. 假设数据库迁移脚本已版本控制（alembic）
5. 假设测试数据工厂可复用现有 CustomerFactory + UserFactory

#### 依赖

1. **Celery 测试基础设施** - 预实现里程碑前需要（BLK-01）
2. **HTTP Mock 基础设施** - 预实现里程碑前需要（BLK-02）
3. **数据权限测试支持** - 预实现里程碑前需要（BLK-03）
4. **pytest-benchmark 性能测试框架** - Sprint 1 需要
5. **Pact.io 契约测试框架** - Sprint 1 需要

#### 计划风险

- **风险**: pytest-benchmark 与现有 pytest 配置兼容性问题
  - **影响**: 性能基准测试可能无法运行
  - **应急计划**: 使用替代方案（pytest-timeout + 手动计时）

- **风险**: Pact.io 学习曲线陡峭
  - **影响**: 契约测试实施延迟
  - **应急计划**: 先用 pytest-httpx mock，后补契约测试
