# Product Brief: 内部运营中台客户信息管理与运营系统

## Executive Summary

**核心问题：** 客户数据分散在 Excel 中，1320 个客户缺乏统一管理，导致运营效率低下和收入损失。

**业务影响：** 
- 运营团队每月花费 3 天手工生成结算单，且数据不准确
- 客户流失风险不可控（193 个僵尸账号未被识别）
- 93.6% 客户未评级，无法进行精细化运营
- 收入预测不准确，影响业务决策

**解决方案愿景：** 建设统一的客户信息管理与运营中台，实现客户数据集中管理、账号自动治理、客户价值自动分层、结算自动化，提升运营效率和决策准确性。

**核心价值：**
- 结算单生成时间从 3 天缩短到分钟级
- 100% 客户自动分层，支撑精细化运营
- 僵尸账号自动识别和预警，降低流失风险
- 统一数据视图，提升收入预测准确性

**关键差异化：**
- 数据驱动：基于 1320 条真实客户数据设计的系统
- 中台架构：能力复用，避免重复建设
- 开放集成：预留 API 支持外部系统对接
- 分阶段实施：先解决最紧迫问题，快速见效

## Core Vision

### Problem Statement

**核心问题：** 客户数据管理落后于业务发展，手工 Excel 管理方式无法支撑 1320+ 客户规模的精细化运营需求。

**具体表现：**
1. **数据分散** - 客户信息、结算数据、用量数据分散在不同 Excel 和系统中，缺乏统一视图
2. **效率低下** - 运营团队每月花费 3 天手工生成结算单，占用大量人力资源
3. **准确性差** - 手工操作导致数据错误，结算单准确性无法保证
4. **风险不可控** - 193 个近一年未使用的"僵尸账号"未被主动识别，客户流失风险高
5. **运营粗放** - 93.6% 客户未进行等级评定，无法实现差异化运营
6. **决策困难** - 缺乏实时数据支持，收入预测不准确，影响业务决策

### Problem Impact

**业务影响量化：**

| 影响维度 | 具体表现 | 业务损失 |
|---------|---------|---------|
| 运营效率 | 每月 3 天手工对账 | 运营人力浪费，无法聚焦高价值工作 |
| 收入损失 | 僵尸账号未被识别 | 193 个潜在流失客户，收入损失风险 |
| 客户体验 | 结算单错误 | 客户信任度下降，合作风险增加 |
| 决策质量 | 收入预测不准确 | 业务规划偏差，资源投入失误 |
| 风险管理 | 坏账风险不可控 | 财务风险累积，现金流影响 |

**如果不解决，6-12 个月后的后果：**
- 客户规模持续增长，Excel 管理方式彻底失效
- 运营团队人力瓶颈凸显，无法支撑业务增长
- 客户流失率上升，收入增长放缓
- 数据质量问题累积，系统迁移成本更高
- 竞争对手通过数字化运营获得优势

### Why Existing Solutions Fall Short

**当前解决方案（Excel）的致命缺陷：**

| 缺陷维度 | Excel 管理方式 | 业务影响 |
|---------|--------------|---------|
| 数据规模 | 1320 条记录已超出 Excel 高效管理范围 | 打开慢、编辑卡顿、易崩溃 |
| 数据一致性 | 多人编辑导致版本混乱 | 数据冲突，难以合并 |
| 自动化 | 完全手工操作 | 每月 3 天重复劳动 |
| 预警能力 | 无主动预警机制 | 僵尸账号、坏账风险无法提前识别 |
| 数据完整性 | 90% 字段缺失率 | 无法支撑精细化运营 |
| 权限管理 | 缺乏细粒度权限控制 | 数据安全风险 |
| 系统集成 | 信息孤岛 | 无法与财务/销售系统对接 |
| 数据分析 | 分析能力有限 | 难以支撑深度业务洞察 |

**为什么现有 CRM 无法直接使用：**
- 通用 CRM 缺乏房产行业特定功能
- 定价模型复杂（单层/多层/阶梯/包年），标准 CRM 不支持
- 需要与内部其他系统深度集成
- 成本高昂，实施周期长

### Proposed Solution

**内部运营中台客户信息管理与运营系统**

**系统定位：** 面向房产行业的客户信息管理与运营一体化平台，整合客户管理、结算管理、账号治理、画像管理四大核心能力。

**核心功能：**

**第一期（3.5 个月）：**
1. **客户主数据管理 (MDM)** - 1320 个客户统一视图，结束数据分散
2. **客户健康度监控** - 僵尸账号自动识别，活跃度预警
3. **客户价值评估** - 自动分层（S/A/B/C/D），支撑精细化运营

**第二期（6 个月）：**
4. **统一价格中心** - 多定价策略管理（单层/多层/阶梯/包年/免费 + 超费）
5. **数据采集中台** - 用量数据自动采集，告别手工录入
6. **智能结算引擎** - 结算单自动生成，从 3 天缩短到分钟级
7. **风控预警系统** - 信用评估、回款预警、催收管理
8. **经营分析 BI** - 管理仪表盘，收入预测、经营分析

**技术架构：**
- 后端：Python 3.11 + Sanic（异步高性能）
- 前端：Vue 3 + Arco Design（现代化 UI）
- 数据库：PostgreSQL 18（成熟稳定）
- 部署：Docker 容器化（快速部署、弹性扩展）

**实施策略：**
- 分阶段：先解决最紧迫问题（数据治理），再扩展能力
- 快速迭代：3.5 个月上线第一期，快速验证价值
- 开放集成：预留 API，支持未来与外部系统对接

### Key Differentiators

**核心竞争优势：**

| 差异化维度 | 传统 CRM/自建方案 | 本系统方案 | 业务价值 |
|-----------|-----------------|-----------|---------|
| 行业适配 | 通用功能，需大量定制 | 房产行业深度定制 | 开箱即用，降低实施风险 |
| 定价模型 | 简单定价 | 支持复杂定价（单层/多层/阶梯/包年） | 适应业务多样性 |
| 数据驱动 | 基于最佳实践 | 基于 1320 条真实数据设计 | 精准匹配业务需求 |
| 实施周期 | 6-12 个月 | 9 个月（2 期） | 快速见效，降低风险 |
| 技术栈 | 可能过度设计 | Python+Vue，轻量高效 | 降低运维成本 |
| 集成能力 | 封闭系统 | 开放 API，支持外部集成 | 保护现有投资 |
| 分阶段交付 | 一次性交付 | 3.5 个月上线第一期 | 快速验证，灵活调整 |

**为什么现在是最佳时机：**
1. **业务规模临界点** - 1320 个客户，Excel 管理已到极限
2. **数据积累充分** - 有足够的历史数据支撑系统设计
3. **团队共识形成** - 四大角色痛点明确，需求清晰
4. **技术成熟度高** - Python/Vue/PostgreSQL 技术栈成熟，降低技术风险
5. **竞争压力增加** - 行业数字化加速，需要提升运营效率保持竞争力

**独特洞察：**
- **房产行业客户管理特殊性：** 客户类型多样（项目/经纪/ERP）、定价复杂、结算周期长，通用 CRM 无法直接满足
- **数据治理先行：** 先解决数据质量问题，再谈智能化，避免"垃圾进垃圾出"
- **中台架构思维：** 能力复用，避免重复建设，支撑未来业务扩展

### MVP 聚焦策略（Party Mode 决策）

**第一期聚焦运营角色：**

| 维度 | 决策 |
|------|------|
| **核心用户** | 运营团队（2-3 人） |
| **核心场景** | 客户查询 → 健康度查看 → 分层查看 → 数据编辑 |
| **关键指标** | 客户查询时间从 10 分钟 → 30 秒 |
| **砍掉的功能** | 结算管理、财务对账、管理层 BI（移至第二期） |

**第一期功能优先级（仅运营视角）：**
1. 客户 360 视图（查询 + 编辑）
2. 健康度仪表盘（僵尸账号识别）
3. 客户分层展示（S/A/B/C/D）
4. 基础数据导入（一次性脚本）

**数据迁移策略：**
- **方案：** 全量清洗 + 一次性脚本导入，暂不开发批量导入功能
- **清洗规则：**
  - 必填字段：公司 id、公司名称、账号类型
  - 可选字段：行业类型、客户等级、合作状态
  - 忽略字段：所属 ERP、接入时间（缺失率太高）
- **实施：** Python 脚本读取 Excel → 清洗 → 写入 PostgreSQL

**产品化定位：**
- **决策：** 不做外部产品化，聚焦内部系统
- **差异化聚焦：**
  1. 极度贴合业务：复杂定价模型支持
  2. 快速响应变化：内部系统优势是迭代快
  3. 零学习成本：UI/UX 完全贴合现有工作习惯
- **内部 SLA 建议：** 响应时间<2 秒，可用性 99%，数据准确性 99.9%

### ROI 分析（Party Mode 测算）

**成本估算：**
- 开发人力：1 个全栈×9 个月 ≈ 45 万（假设月薪 5 万）
- 服务器成本：约 500 元/月 × 12 = 6000 元/年
- **总计：** 首年约 45.6 万

**收益测算（仅运营效率）：**
- 每月 3 天结算时间 → 假设 2 人天，每月节省 3000 元
- 僵尸账号识别：挽回 10% 流失客户，约 19 个 × 500 元/月 = 9500 元/月
- **年度收益：** (3000+9500) × 12 = 15 万/年

**投资回收期：** 45.6 万 ÷ 15 万/年 ≈ 3 年

**注意：** 此测算偏保守，未包含决策质量提升、客户满意度提升、运营团队高价值工作等隐性收益。

## Target Users

### Primary Users

#### 👤 用户画像 1: 运营经理 - "张经理"

**基本信息：**
- 角色：运营经理
- 职责：统筹客户运营工作，管理运营专员，对运营效率和结果负责
- 技术能力：能够接受新系统，有基础的数据分析能力

**核心 KPI：**
- 结算及时性：每月月初按时完成结算单生成和发送
- 总收入金额：监控客户消费情况，确保收入目标达成

**核心需求：**
- 全局视图：快速了解所有客户的运营状况
- 数据分析：用量趋势、消费趋势、客户分层分布
- 异常监控：及时发现用量异常、结算延迟、客户流失风险
- 决策支持：基于数据做出客户分级、资源投入决策

**痛点场景：**
> 每月 1-3 号是结算高峰期，张经理需要加班到深夜，因为要从 Excel 中汇总所有客户的用量，手动计算费用，生成结算单。不仅耗时 3 天，还容易出错。如果数据有误，还需要重新和客户对账，影响客户满意度。

**成功时刻：**
> 系统上线后，每月 1 号上午 9 点，一键生成所有客户的结算单，准确率 100%，2 小时内完成发送。张经理可以从容地分析运营数据，制定客户运营策略，而不是被重复劳动困住。

#### 👤 用户画像 2: 运营专员 - "小李"

**基本信息：**
- 角色：运营专员
- 职责：日常客户沟通、数据维护、结算单发送、客户咨询处理
- 技术能力：Excel 熟练，对 CRM 系统接受度高

**核心 KPI：**
- 客户用量：负责的客户群体用量增长情况
- 客户消费：负责的客户群体消费金额达成情况

**日常工作：**
1. 客户咨询处理：回答客户关于用量、费用、合同的咨询
2. 数据维护：更新客户信息、合同信息、定价信息
3. 结算支持：协助生成结算单，发送客户，处理异议
4. 用量监控：查看客户用量趋势，发现异常及时沟通

**核心需求：**
- 快速查询：30 秒内找到客户完整信息
- 数据编辑：快速更新客户信息，无需复杂流程
- 用量查看：实时查看客户用量和消费情况
- 预警提醒：客户用量异常、合同到期自动提醒

**痛点场景：**
> 销售同事打电话问："某某客户这个月用了多少？"小李需要在多个 Excel 文件中查找，花 10 分钟才能找到数据。如果客户问"我的费用怎么算的？"需要手动重新计算，耗时且容易出错。

**成功时刻：**
> 销售同事在系统中直接查看客户用量，不再需要反复询问。小李每天可以节省 2 小时查询时间，专注于客户沟通和运营策略，而不是当"人肉搜索引擎"。

### User Journey

#### 🎬 场景 A: 新客户接入

**参与者：** 运营专员

**流程：**
1. 销售签单后，通知运营创建客户档案
2. 运营在系统中创建客户基础信息
3. 设置客户定价策略（单层/多层/阶梯/包年）
4. 配置客户账号和权限
5. 开始计量用量

**成功指标：**
- 创建时间：5 分钟内完成
- 准确性：100% 信息准确
- 及时性：签约当天完成配置

---

#### 🎬 场景 B: 月末结算（核心痛点场景）

**参与者：** 运营经理、运营专员

**时间：** 每月月初（结算上个月全月用量）

**当前流程（手工）：**
1. 月末从各系统导出用量数据（耗时 2 小时）
2. 在 Excel 中汇总整理（耗时 4 小时）
3. 根据定价手动计算费用（耗时 6 小时）
4. 生成结算单并发送给客户（耗时 4 小时）
5. 处理客户异议和对账（耗时 8 小时）
6. 确认收款并更新状态（耗时 4 小时）
**总计：约 3 个工作日，易出错**

**目标流程（系统化）：**
1. 系统自动采集用量数据（自动）
2. 系统根据定价自动计算（自动）
3. 一键生成结算单，运营审核（30 分钟）
4. 批量发送结算单给客户（自动）
5. 客户异议处理（系统支持）
6. 收款确认并更新状态（简化）
**总计：约 2 小时，准确率 100%**

**成功指标：**
- 结算时间：从 3 天 → 2 小时
- 准确率：从 90% → 100%
- 客户满意度：投诉率降低 80%

---

#### 🎬 场景 C: 销售查询客户

**参与者：** 销售团队

**流程：**
1. 销售登录系统
2. 搜索客户（支持公司名、账号类型等）
3. 查看客户详情页：
   - 基础信息
   - 历史用量趋势
   - 消费金额趋势
   - 合同信息
4. 导出或截图用于拜访准备

**成功指标：**
- 查询时间：<30 秒
- 数据完整性：100% 信息可查
- 移动端支持：手机可访问

### Future Vision

#### 第二期（6 个月）：结算自动化

**目标：** 实现结算全流程自动化，从 3 天缩短到 2 小时

**核心功能：**
- **智能结算引擎：** 自动对账、结算单生成、批量发送
- **统一价格中心：** 单层/多层/阶梯/包年/免费 + 超费多种定价模型
- **数据采集中台：** 用量数据自动采集，支持多数据源对接
- **风控预警系统：** 信用评估、回款预警、催收管理
- **经营分析 BI：** 管理仪表盘、收入预测、经营分析报告

**业务价值：**
- 结算时间：3 天 → 2 小时（92% 提升）
- 结算准确率：90% → 100%
- 运营人力节省：50%+

#### 第三期（扩展期）：平台化

**目标：** 扩展移动端和开放平台能力

**核心功能：**
- **移动端 App：** iOS/Android，支持销售外出查看客户
- **开放 API 平台：** 支持外部系统调用（结算单查询/状态更新）
- **客户自助门户：** 客户登录后查看用量、结算单、发起咨询
- **AI 智能分析：** 用量预测、流失预警、智能推荐

**业务价值：**
- 销售效率：查询时间 10 分钟 → 30 秒，随时随地访问
- 客户体验：自助服务，减少沟通成本
- 生态价值：开放集成，保护现有 IT 投资

#### 长期愿景（2-3 年）

**目标：** 成为房产行业领先的客户管理与运营平台

**战略方向：**
- **房产行业客户管理 SaaS：** 可选产品化，服务同行企业
- **生态集成平台：** 对接房产 ERP、财务系统、CRM 等第三方系统
- **数据智能平台：** 行业洞察、市场趋势分析、对标分析

**商业价值：**
- 从成本中心到利润中心（如产品化）
- 行业影响力和话语权提升
- 数据驱动的业务创新和增长
