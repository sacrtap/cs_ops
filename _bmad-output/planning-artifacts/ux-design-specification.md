## Executive Summary

### Project Vision

**内部运营中台客户信息管理与运营系统** 是一个企业内部 Web 应用，旨在解决客户数据分散、运营效率低下、客户流失风险高的核心问题。

**核心痛点：**
- 1320 个客户数据分散在 Excel 中，缺乏统一管理
- 运营团队每月花费 **3 天** 手工生成结算单，数据不准确
- **193 个僵尸账号**（14.6%）未被识别，客户流失风险高
- **93.6% 的客户** 未进行价值评级，无法精细化运营

**解决方案愿景：** 建设统一的客户信息管理与运营中台，实现客户数据集中管理、账号自动治理、客户价值自动分层、结算自动化，提升运营效率和决策准确性。

**核心价值量化：**
- 结算单生成时间：从 3 天 → 2 小时（92% 提升）
- 僵尸账号识别率：从 0% → 100%
- 客户价值评级覆盖率：从 6.4% → 100%
- 客户查询响应时间：从 10 分钟 → 30 秒（95% 提升）

### Target Users

| 用户角色 | 核心职责 | 关键需求 | 使用频度 |
|---------|---------|---------|---------|
| **👤 运营经理 - 王芳** | 配置定级规则、查看数据分析、管理客户分配 | 全局视图、数据分析、异常监控、决策支持 | 每日查看仪表盘，每周配置规则 |
| **👤 运营专员 - 李明/小王** | 日常客户数据维护、结算单生成、客户转移操作 | 快速查询、数据编辑、批量操作、预警处理 | 高频使用，每日处理客户咨询和维护 |
| **👤 销售团队 - 张伟** | 查看自己负责的客户信息、了解客户价值等级 | 客户信息查询、用量查看、移动访问 | 按需使用，拜访前准备和响应客户咨询 |

**权限模型：**
- **Admin**：系统全部功能
- **经理**：查看全部客户、配置规则、数据分析
- **专员**：查看全部客户、数据维护、结算操作
- **销售**：仅查看自己负责的客户，无编辑权限

### Key Design Challenges

**1. 复杂数据简化展示**
- 1320 个客户数据的高效列表展示和筛选
- 客户 360 度视图的信息组织，避免信息过载
- 多维度数据（基础信息/用量/结算/健康度）的层次化呈现

**2. 多层级权限的界面适配**
- 4 级权限下的统一界面设计
- 销售角色只能查看自己客户的权限边界清晰展示
- 不同角色看到的功能入口和数据范围差异化

**3. 数据可视化与可读性平衡**
- 健康度仪表盘的多指标卡片式布局
- 用量趋势、消费趋势的图表简洁展示
- 运营报告的一键生成和导出

**4. 移动端销售场景优化**
- 外出拜访时的快速客户信息查询
- 手机屏幕上的信息优先级排序
- 一键导航、点击拨号等移动场景支持

### Design Opportunities

**1. 智能化搜索与筛选**
- 支持多维度组合筛选（价值等级/行业/状态/销售负责人）
- 智能搜索建议和历史记录
- 高级筛选条件保存和复用

**2. 预测性预警设计**
- 僵尸账号、风险客户主动推送预警
- 仪表盘关键指标异常颜色标记
- 预警分级处理（重要/一般/提示）

**3. 批量操作优化**
- 批量导入/导出的进度可视化和错误处理
- 批量发送结算单的状态跟踪
- 减少重复操作，提升运营效率

**4. 数据可追溯性**
- 客户转移历史记录完整展示
- 定级变更原因记录 and 追溯
- 操作日志的审计视图

## Core User Experience

### Defining Experience

**核心体验定义：** 内部运营中台客户信息管理与运营系统的核心体验是 **"30 秒客户信息可达"** - 任何用户（销售/运营/经理）在需要客户信息时，通过全局搜索和智能筛选，在 30 秒内获得完整、准确的客户 360 度视图。

**核心价值主张：**
- 销售：快速响应客户，展现专业性
- 运营：自动化结算，告别手工加班
- 经理：数据驱动决策，精细化运营

**业务流程影响（ROI 分析）：**

| 角色 | 当前耗时 | 目标耗时 | 提升 | 年化价值 |
|------|---------|---------|------|---------|
| 销售查询 | 10 分钟/次 × 5 次/周 | 30 秒/次 | 95%↓ | ~200 小时/年 |
| 运营结算 | 3 天/月 | 2 小时/月 | 92%↓ | ~300 小时/年 |
| 经理报表 | 4 小时/周 | 30 分钟/周 | 87%↓ | ~180 小时/年 |
| **总计** | | | | **~680 小时/年** |

*假设人力成本 100 元/小时，年化价值 = 6.8 万元*

### Platform Strategy

**平台策略：内部 Web 应用（多端适配）**

- **主平台**：PC Web 端（1920px→1366px）- 运营团队日常工作
- **移动端**：响应式 Web（375px→768px）- 销售外出拜访
- **交互方式**：PC 端支持键盘快捷键，移动端支持触屏手势
- **技术栈**：Vue 3 + Arco Design + TypeScript，响应式布局

**架构决策：**
- ✅ 前端搜索 + 后端索引混合方案（平衡速度和灵活性）
- ✅ 标签页懒加载（优化首屏性能）
- ✅ 权限在 API 层统一拦截（安全优先）
- ⚠️ PWA 离线功能：MVP 不需要，未来可考虑

### Effortless Interactions

**无需思考的交互设计：**

1. **全局搜索** - 输入即搜索，支持姓名/电话/公司模糊匹配，防抖 300ms
2. **智能筛选** - 常用条件一键切换（我的客户/高价值/风险/僵尸）
3. **批量操作** - 导入/导出/发送结算单，进度可视化和错误处理
4. **预警处理** - 仪表盘红色标记，点击直达处理页面
5. **移动端一键拨号** - `<a href="tel:{{phone}}">` 直接呼叫

**当前用户痛点（竞品对比）：**
- ❌ 传统 CRM：多层导航才能找到客户详情
- ❌ 手工 Excel：筛选功能弱，无法实时协作
- ✅ **我们的方案**：全局搜索 + 智能筛选 + 一键直达

### Critical Success Moments

**关键成功时刻：**

1. **销售 30 秒查询** - 客户来电→输入名字→看到完整信息
   - 场景：高压力、时间敏感
   - 成功标准：无需培训，直觉操作

2. **运营 2 小时结算** - 选择月份→一键生成→异常高亮→批量发送
   - 场景：高容量、重复性操作
   - 成功标准：提前完成，无加班

3. **经理 5 分钟配置** - 调整阈值→保存生效→预览影响客户数
   - 场景：探索性、决策支持
   - 成功标准：无需技术介入，业务人员自主配置

**失败即毁体验：**
- ❌ 搜索找不到客户（需支持模糊搜索和别名）
- ❌ 结算单数据错误（需异常检测和人工复核）
- ❌ 权限混乱（销售看到别人客户）

### Edge Cases to Address

- **网络不稳定场景**：销售在电梯/地下室查询，需要 loading 状态和错误提示
- **月底高峰期**：运营批量操作时系统响应可能变慢，需要进度提示
- **一键导出美化报告**：经理向 CEO 汇报时需要可打印格式
- **并发冲突**：多人同时编辑同一客户数据，需要乐观锁处理
- **导入格式错误**：Excel 格式不规范，需要清晰的错误定位和修正建议

## Desired Emotional Response

### Primary Emotional Goals

**内部运营系统的核心情感目标：**

对于企业内部运营系统，情感设计的核心不是"惊喜"和"娱乐"，而是**工作效率、数据可信度和操作掌控感**。

**5 大核心情感目标：**

| 情感 | 描述 | 为什么重要 |
|------|------|------------|
| **😌 自信 (Confidence)** | 用户确信自己在做正确的事 | 快速决策，减少犹豫和反复确认 |
| **🛡️ 信任 (Trust)** | 用户相信数据和系统的可靠性 | B 端系统的数据准确性是生命线 |
| **🎮 掌控 (Control)** | 用户知道系统在做什么，可随时干预 | 减少焦虑，提升安全感 |
| **⚡ 高效 (Efficiency)** | 用户感觉时间被有效利用 | 直接转化为业务价值和用户满意度 |
| **🎉 成就 (Accomplishment)** | 用户完成工作后有满足感 | 提升工作动力，减少职业倦怠 |

**销售团队 - 客户来电场景：**

| 阶段 | 场景 | 当前情感 | 目标情感 | 设计支持 |
|------|------|----------|----------|----------|

| 出错 | 网络不好/找不到 | 😠 愤怒 | 😐 理解 | 友好错误提示 + 解决方案 |

**运营专员 - 月度结算场景：**

| 阶段 | 场景 | 当前情感 | 目标情感 | 设计支持 |
|------|------|----------|----------|----------|

|------|------|----------|----------|----------|

### Micro-Emotions

**关键微情感识别：**

| 微情感对 | 选择 | 为什么重要 | 设计支持 |
|----------|------|------------|----------|
| 自信 vs 困惑 | **自信** | 用户需要快速做决策 | 清晰视觉层次、明确导航路径 |
| 信任 vs 怀疑 | **信任** | B 端系统数据准确性是生命线 | 数据来源标注、计算逻辑透明 |
| 掌控 vs 焦虑 | **掌控** | 运营需要知道系统在做什么 | 进度可视化、状态反馈、可撤销 |
| 高效 vs 挫败 | **高效** | 时间是核心成本 | 快捷键、批量操作、智能预测 |
| 安心 vs 担心 | **安心** | 月底结算不容有失 | 异常检测、人工复核、备份恢复 |
| 成就 vs 疲惫 | **成就** | 完成工作后的满足感 | 完成庆祝、效率报告、准时下班 |

### Design Implications

**情感 1：自信 → 清晰层次 + 即时反馈**
- 关键信息用大字体、粗体、颜色突出
- 面包屑导航、当前位置高亮
- 点击后 100ms 内给出视觉反馈
- 一致的设计语言，建立心智模型

**情感 2：信任 → 数据透明 + 异常提示**
- 显示"数据更新于：XXX"
- 定级规则旁显示"计算规则：XXX"
- 异常主动提示："检测到异常：XXX"
- 错误友好提示：说明原因和解决方案

**情感 3：掌控 → 进度可视 + 可撤销**
- "正在处理... 45/100 (45%)"
- "已发送 80 封，失败 2 封，待发送 18 封"
- 支持撤回操作（10 分钟内）
- 批量操作前预览："即将发送 100 封，预计 5 分钟"

**情感 4：高效 → 快捷键 + 智能预测**
- 键盘快捷键：Ctrl+F 搜索、Ctrl+S 保存、Esc 关闭
- 输入名字自动显示最近联系的客户
- 选择多个客户→批量导出/发送
- 常用操作在 2 次点击内完成

**情感 5：成就 → 完成庆祝 + 效率报告**
- "✅ 完成！节省了 2.8 小时"
- "本月您处理了 1320 个客户，比去年快 92%"
- "连续 3 个月准时完成结算！"
- "太棒了！您可以准时下班了 🎉"

### Emotional Design Principles

**指导原则：**

| 原则 | 描述 | 设计指导 |
|------|------|----------|
| **🎯 清晰优于美观** | B 端系统，信息清晰度优先于视觉华丽 | 用对比度、字体大小、颜色区分优先级 |
| **📊 透明建立信任** | 数据来源、计算逻辑、操作记录完全透明 | 每个数据都可追溯，每个操作都有日志 |
| **⚡ 速度创造自信** | 快速响应让用户感觉系统可靠 | 搜索<500ms，加载<2s，点击即反馈 |
| **🔒 掌控减少焦虑** | 进度可视、状态清晰、可撤销 | 批量操作进度条，重要操作二次确认 |
| **🎉 庆祝提升动力** | 完成关键任务后给予正向反馈 | 结算完成显示效率提升，准时下班鼓励 |

## UX Pattern Analysis & Inspiration

### Inspiring Products Analysis

**销售团队常用应用：**
- **企业微信/钉钉** - 即时通讯 + 客户管理集成，一键拨号
- **手机通讯录** - 一键拨号，通话记录自动保存
- **Salesforce Mobile** - 移动端客户查询方便

**运营团队常用应用：**
- **Excel/Google Sheets** - 灵活筛选、批量操作、公式计算
- **Outlook** - 批量发送、邮件模板、发送状态跟踪
- **飞书文档** - 多人协作、版本历史、一键导出

**运营经理常用应用：**
- **Tableau/Power BI** - 数据可视化、仪表盘、趋势分析
- **钉钉管理后台** - 审批流程、数据分析、团队管理
- **飞书 OKR** - 目标追踪、进度可视化

### Transferable UX Patterns

**1. 全局搜索（借鉴 Google/钉钉）**
- 顶部固定搜索栏，输入即搜索
- 搜索结果实时反馈（防抖 300ms）
- 精确匹配优先，关键字段高亮
- 搜索历史自动保存
- 快捷键 `/` 快速聚焦

**2. 智能筛选（借鉴 Excel/飞书表格）**
- 常用筛选一键切换（我的客户/高价值/风险/僵尸）
- 多条件组合筛选（等级 + 行业 + 状态 + 负责人）
- 筛选条件标签化展示，可一键清除
- 筛选结果数量实时显示
- 支持保存筛选方案

**3. 卡片式仪表盘（借鉴 Tableau/钉钉管理后台）**
- 关键指标卡片化，一眼可见
- 异常数据红色/黄色标记
- 趋势图表简洁明了
- 时间范围快速切换（今天/本周/本月）
- 卡片布局可自定义

**4. 批量操作（借鉴 Outlook/Excel）**
- 复选框多选，支持全选
- 选中后显示批量操作栏
- 进度可视化（XX/YY，百分比）
- 错误处理和重试机制
- 操作结果报告（成功/失败明细）

**5. 标签页分层（借鉴浏览器/Settings）**
- 信息分层，避免单页过载
- 标签清晰命名（基础/用量/结算/跟进/历史）
- 切换无刷新，懒加载数据
- 键盘快捷键切换（Ctrl+1~5）

**6. 一键拨号（借鉴手机通讯录）**
- 电话号码可点击（`<a href="tel:">`）
- 移动端直接呼叫
- PC 端支持复制号码
- 通话记录自动保存

**7. 进度可视化（借鉴文件上传）**
- 进度条实时百分比
- 剩余时间估算
- 当前处理项显示
- 支持暂停/取消
- 完成报告汇总

**8. 错误友好提示（借鉴 Stripe）**
- 清晰说明错误原因
- 提供具体解决方案
- 定位到错误位置
- 支持一键修复（如果可能）
- 语气友好，不指责用户

### Anti-Patterns to Avoid

**1. 多层导航迷宫**
- ❌ 避免：首页→客户管理→客户列表→搜索→客户详情
- ✅ 应该：全局搜索→直接到客户详情

**2. 权限错误惊吓**
- ❌ 避免：销售点击"查看全部客户"→弹出"403 无权访问"
- ✅ 应该：销售天然只看到"我的客户"菜单

**3. 批量操作无反馈**
- ❌ 避免：点击"批量发送"→页面卡住 30 秒→无提示
- ✅ 应该：进度条实时显示，剩余时间估算，支持取消

**4. 数据更新不同步**
- ❌ 避免：运营修改客户信息→销售看到旧数据
- ✅ 应该：数据变更实时推送，或标注"最后更新：10 分钟前"

**5. 移动端体验阉割**
- ❌ 避免：移动端只能查看列表
- ✅ 应该：移动端完整支持查询、详情、一键拨号

### Design Inspiration Strategy

**直接采用（成熟模式）：**
- ✅ 全局搜索（顶部固定，输入即搜索）
- ✅ 标签页分层（客户 360 度视图）
- ✅ 一键拨号（移动端`<a href="tel:">`）
- ✅ 卡片式仪表盘（关键指标 + 异常标记）

**适配修改（根据业务需求）：**
- 🔧 智能筛选：增加"我的客户/高价值/风险"业务筛选
- 🔧 批量操作：增加异常检测和人工复核
- 🔧 进度可视化：增加业务含义（"已生成 45/100 个结算单"）

**绝对避免（与目标冲突）：**
- ❌ 多层导航迷宫（违背"30 秒可达"）
- ❌ 权限错误惊吓（违背"权限透明"）
- ❌ 批量操作无反馈（违背"掌控"情感）
- ❌ 数据不透明（违背"信任"情感）

## Defining Experience

### The Core Interaction: "输入即得"

**定义性体验：** 销售/运营输入客户姓名/电话，系统在 500ms 内呈现完整、准确的客户 360 度视图。

**为什么这是核心：**
- 最高频场景（销售每日多次，运营每日使用）
- 其他所有操作的前提（找不到客户，后续操作无从谈起）
- 直接解决最大痛点（从 10 分钟 → 30 秒）
- 用户会向同事炫耀的功能（"超快，一搜就到"）

### User Mental Model

**用户当前做法：**
1. 打开 Excel 表格 → 多个文件，不知道哪个最新
2. Ctrl+F 查找 → 1320 行数据，查找慢，怕客户等待
3. 手工核对 → 需要打开多个表格（基础/用量/结算）
4. 打电话问同事 → 显得不专业

**用户心智模型：**
- 📇 通讯录思维 - 输入名字/电话 → 看到联系人详情
- 📊 Excel 思维 - 列表展示，支持筛选和排序
- 📱 移动端思维 - 一键拨号，点击即呼叫

**用户期望：**
- ⚡ 即时响应 - 输入后立即看到结果
- 🎯 准确匹配 - 找到正确的客户（支持模糊搜索）
- 📋 完整信息 - 所有信息在一个页面

### Success Criteria

**核心体验成功标准：**

| 指标 | 目标值 | 用户感受 |
|------|--------|----------|
| 搜索响应时间 | < 500ms | "哇，好快！" |
| 搜索结果准确率 | > 99% | "一搜就准！" |
| 首次点击准确率 | > 90% | "不用试错，一次就找到" |
| 信息完整度 | 100% | "所有信息都在一个页面" |
| 权限透明度 | 0 权限错误 | "自然看到我能看的" |

**用户感到聪明的时刻：**
- 输入客户简称（"腾迅"）也能找到"腾讯科技"
- 输入手机号，自动格式化并匹配客户
- 搜索结果高亮显示匹配字段，一眼确认

**自动完成的魔法：**
- 输入时实时搜索，不用点击"搜索"按钮
- 自动保存搜索历史，快速回访
- 销售自动只看到自己的客户，无需额外操作

### Novel vs. Established Patterns

**既有模式（用户已熟悉）：**
- 🔍 全局搜索 - Google/钉钉/企业微信都在用
- 📋 列表筛选 - Excel/飞书表格的筛选功能
- 📇 联系人详情 - 手机通讯录的详情页面

**我们的创新组合：**
- 🎯 业务智能搜索 - 搜索名字/价值等级/状态/负责人
- 🎨 权限感知筛选 - 销售天然只看到自己客户
- 📊 360 度视图标签页 - 基础/用量/结算/跟进/历史分层

**如何教导用户：**
- 渐进式披露 - 先看到基础信息，点击标签页看详情
- 视觉引导 - 价值等级用颜色标记（S 级红色醒目）
- 空状态提示 - 新用户第一次搜索显示引导文案

### Experience Mechanics

**核心交互："输入即得"搜索体验**

**1. 启动（Initiation）：**
- 顶部搜索栏始终可见（固定在导航栏）
- 按 `/` 键快速聚焦搜索框（全局快捷键）
- placeholder 文案："搜索客户姓名/电话/公司，按 `/` 快速搜索"

**2. 交互（Interaction）：**
- 输入文字（姓名/电话/公司）
- 实时显示搜索结果（防抖 300ms）
- 点击搜索结果进入详情页
- 系统响应：< 500ms 渲染完成，关键字段高亮

**3. 反馈（Feedback）：**
- Loading 状态：搜索中显示旋转图标（< 200ms）
- 结果计数："找到 3 个匹配客户"
- 匹配高亮：搜索词在结果中黄色高亮
- 空状态："未找到客户，试试其他关键词"
- 错误处理：网络异常/无结果/权限问题友好提示

**4. 完成（Completion）：**
- 点击搜索结果 → 跳转到客户详情页
- 详情页显示 4 个标签页（基础/用量/结算/跟进）
- 下一步：一键拨号/编辑/导出/继续搜索

### Journey Patterns

跨旅程识别的可复用设计模式：

**1. 导航模式：全局搜索 + 标签页详情**
- **模式描述**：用户通过全局搜索快速定位目标，通过标签页分层查看详情
- **应用场景**：客户查询、结算单查询、用量查询
- **设计规范**：
  - 搜索响应 < 500ms
  - 详情页标签页 3-5 个
  - 标签页懒加载数据

**2. 批量操作模式：选择→预览→确认→进度→报告**
- **模式描述**：5 步标准化批量操作流程
- **应用场景**：批量发送结算单、批量导入客户、批量导出
- **设计规范**：
  - 选择后显示已选数量
  - 预览支持编辑
  - 进度实时百分比显示
  - 完成报告包含成功/失败统计

**3. 异常处理模式：主动检测→分类展示→快速修正**
- **模式描述**：系统主动检测异常，分类展示，提供快速修正入口
- **应用场景**：结算异常、用量异常、价格缺失
- **设计规范**：
  - 异常颜色标记（警告橙/危险红）
  - 每条异常提供 2-3 个快速操作
  - 支持批量修正

**4. 仪表盘模式：关键指标→趋势图表→预警列表**
- **模式描述**：3 层信息架构，从概览到细节
- **应用场景**：运营仪表盘、销售仪表盘、财务仪表盘
- **设计规范**：
  - 第一屏：4-6 个关键指标卡片
  - 第二屏：2-3 个趋势图表
  - 第三屏：预警列表（按优先级排序）

**5. 规则配置模式：对比→预览→确认→日志**
- **模式描述**：规则变更前后的影响对比，实时预览，自动记录日志
- **应用场景**：定级规则配置、价格规则配置、预警规则配置
- **设计规范**：
  - 左右对比展示（调整前 vs 调整后）
  - 实时预览影响范围
  - 自动记录变更日志

### Flow Optimization Principles

**1. 最小步骤原则（3 步内完成）**
- 所有核心操作控制在 3 步以内
- 减少不必要的确认步骤
- 默认值智能填充

**2. 渐进披露原则**
- 信息分层展示（概览→详情→明细）
- 避免单页信息过载
- 按需加载数据

**3. 即时反馈原则（500ms 内）**
- 搜索响应 < 500ms
- 页面加载 < 2s
- 点击后 100ms 内视觉反馈

**4. 错误预防原则**
- 主动检测异常（而非事后发现）
- 关键操作二次确认
- 输入验证实时提示

**5. 情感触点原则**
- 识别关键时刻（客户投诉/预警推送）
- 提供情感支持（进度可视化/完成庆祝）
- 正向反馈（效率报告/节省时间）

**6. 一致性原则**
- 相同操作保持一致的交互方式
- 统一的视觉语言（颜色/字体/间距）
- 统一的错误提示格式

### Feedback Patterns

**反馈类型和样式：**

| 类型 | 颜色    | 图标 | 使用场景             | 持续时间 |
| ---- | ------- | ---- | -------------------- | -------- |
| **成功** | `#00B42A` | ✅   | 操作成功完成         | 2 秒     |
| **警告** | `#FF7D00` | ⚠️   | 需要注意但不阻断流程 | 5 秒     |

**反馈形式：**

**Message（轻量提示）：**
- 位置：页面顶部居中
- 尺寸：高度 40px，圆角 4px
- 内容：图标 + 文字（≤50 字）
- 场景：操作成功/失败提示

**Notification（重要通知）：**
- 位置：右上角
- 尺寸：宽度 400px，可堆叠
- 内容：标题 + 详情 + 操作按钮
- 场景：系统通知/预警推送

**Modal（重要确认）：**
- 位置：屏幕居中
- 尺寸：宽度 480px/640px
- 内容：标题 + 详情 + 确认/取消
- 场景：危险操作确认/重要信息展示

**Alert（内联提示）：**
- 位置：嵌入在页面内容中
- 尺寸：全宽/卡片内
- 内容：图标 + 标题 + 详情 + 操作
- 场景：表单错误/异常数据提示

**反馈文案规范：**
```

❌ 不好的：

### Responsive Strategy

**基于用户角色和设备场景的响应式策略：**

**桌面端策略（≥1200px）** ⭐ 主要使用场景（80% 使用时长）
- **目标用户**：运营经理、运营专员
- **布局特征**：顶部导航（60px）+ 左侧边栏（240px，可折叠）+ 主内容区自适应
- **网格系统**：24 列网格，4-5 列卡片布局
- **信息密度**：表格 20-50 条/页，多列布局
- **桌面端特有功能**：批量操作、复杂表单、数据导出、打印功能、键盘快捷键

**平板端策略（768px - 1199px）**
- **目标用户**：运营经理（数据查看）、销售（客户查询）
- **布局调整**：侧边栏折叠为 60px（hover 展开），12 列网格
- **触控优化**：按钮最小 44px × 44px，手势支持（滑动切换标签）
- **表单优化**：双列表单→单列表单，表格支持横向滚动
- **平板端特有功能**：手写输入、语音输入、分屏模式

**移动端策略（375px - 767px）** ⭐ 销售团队主要场景
- **目标用户**：销售团队（外出拜访、客户查询）
- **布局特征**：顶部导航（60px）+ 底部 Tab 导航（60px，3-5 个核心功能）
- **网格系统**：6 列网格，单列垂直布局
- **核心功能**：客户查询、客户详情、一键拨号、创建跟进、导航功能
- **简化功能**：批量操作→单选、复杂表单→分步、数据导出→仅查看、规则配置→仅查看

### Accessibility Strategy

**合规标准：WCAG 2.1 Level AA** ⭐ 行业标准

**选择 Level AA 的理由：**
- ✅ 企业内部系统，包容所有员工（包括残障员工）
- ✅ 法律合规要求（许多国家/地区要求 AA 级别）
- ✅ 行业最佳实践，提升整体 UX 质量
- ✅ 成本效益平衡（AAA 级别成本过高，收益有限）

**视觉无障碍：**

**色彩对比度（符合 WCAG 2.1 AA ≥4.5:1）：**
- 正文文字：#1D2129 on #FFFFFF (对比度 16.1:1) ⭐⭐⭐⭐⭐
- 次要文字：#86909C on #FFFFFF (对比度 4.6:1) ⭐⭐⭐⭐
- 链接文字：#165DFF on #FFFFFF (对比度 4.5:1) ⭐⭐⭐⭐
- 按钮文字：#FFFFFF on #165DFF (对比度 4.5:1) ⭐⭐⭐⭐

**色盲友好设计：**
- 蓝/绿/橙/红区分明显（红绿色盲可识别）
- 不单独依赖颜色传递信息（颜色 + 图标 + 文字三重标识）
- 价值等级：S 级红色 + "S" 文字标签 + 图标
- 成功：🟢 + "成功" 文字，警告：⚠️ + "风险" 文字，危险：🔴 + "僵尸" 文字

**字体缩放支持：**
- 支持浏览器字体缩放 100%-200%
- 使用相对单位 rem（非固定 px）
- 布局自适应，不出现文字截断
- 容器高度自适应，不限制行数
- 最小字号：正文 14px（缩放 200% 后 28px）

**运动无障碍：**

**键盘导航（全键盘操作支持）：**
- Tab 键切换焦点（顺序：搜索框→筛选→列表→分页）
- Enter 键确认（搜索/表单提交）
- Esc 键取消（关闭弹窗/清除搜索）
- 方向键导航（表格内上下移动）
- 全局快捷键：Alt+1~5 导航模块，`/` 聚焦搜索，Ctrl+S 保存，Ctrl+E 导出

**焦点可见：**
- 焦点框：2px solid #165DFF
- 焦点环：offset 2px，确保不与元素重叠
- 焦点顺序：逻辑顺序，从左到右，从上到下

**点击目标尺寸：**
- 桌面端：40px × 40px（Arco 默认）
- 移动端：44px × 44px（iOS Human Interface Guidelines）
- 图标按钮：40px × 40px（图标 20px + 内边距 10px）
- 操作间距：≥8px（避免误触）
- 表格行高：≥40px（桌面）/ ≥48px（移动）

**认知无障碍：**

**清晰的信息架构：**
- 一致的导航模式（顶部导航固定，左侧边栏分类，面包屑显示位置）
- 清晰的标签和说明（使用用户语言，避免技术术语）
- 表单字段提供示例和格式说明
- 错误提示说明原因和解决方案
- 操作按钮文字明确（"保存" 而非 "提交"）

**简化的表单设计：**
- 单列表单优先（避免左右对照）
- 必填字段 ≤ 30%（避免用户压力）
- 分步表单（长表单分组）
- 实时验证（即时反馈，避免提交后报错）
- 自动填充（姓名/电话/地址）

**帮助和引导：**
- 首次使用提示（可跳过）
- 空状态引导（示例 + 操作按钮）
- 工具提示（Hover 显示说明）
- 帮助中心入口（右上角）

### Implementation Guidelines

**响应式开发规范：**

**CSS 最佳实践：**
```css
/* ✅ 使用相对单位 */
body { font-size: 1rem; line-height: 1.5; }
.container { width: 100%; max-width: 1440px; padding: 1rem; }

/* ✅ 移动优先媒体查询 */
.button { width: 100%; padding: 0.75rem; }
@media (min-width: 768px) {
  .button { width: auto; padding: 0.75rem 1.5rem; }
}

/* ✅ 响应式图片 */
img { max-width: 100%; height: auto; }

/* ✅ 响应式网格 */
.grid {
  display: grid;
  gap: 1rem;
  grid-template-columns: 1fr; /* 移动：1 列 */
}
@media (min-width: 768px) {
  .grid { grid-template-columns: repeat(2, 1fr); }
}
@media (min-width: 1200px) {
  .grid { grid-template-columns: repeat(4, 1fr); }
}
```

**Vue 组件响应式：**

<script setup>
import { useBreakpoint } from '@arco-design/web-vue'

```
